History

  • 2006, Christmas – I received a book named Game Programming for Teens, from which I learned Blitz Basic (also known as Blitz 2D)
  • 2007, first half – I learned Python and started doing ProjectEuler problems.
  • 2007, July – I received the sequel named 3D Game Programming for Teens, from which I learned Blitz 3D.
  • 2007, second half – I created my first major programs, one in 2D that would later be called Fractal Mania, and one in 3D called SpaceWars3D, inspired by SpaceWar!.
  • 2010 – I learned LaTeX and some more Java, as well as a little bit of C and Assembly.
  • 2011, summer – I learned seven web programming languages - HTML, CSS, PHP, JavaScript, SQL, XML, and Ajax.
  • 2012, second half – I learned some advanced Java.
  • 2012 – I learned Lua, C, C++, and MIPS Assembly.
  • 2013 – I learned Prolog.
  • 2014, first half – I went through a few CodeAcademy courses, including the ones for Ruby, jQuery, and PHP, within a month.
  • 2014, second half – I did Django's tutorial in a weekend and then spent the next three months developing an employee scheduling website. My skills with Python, Django, HTML, CSS, and JavaScript improved considerably in that time.

Free Time Activities

Small, standalone programs comprise the great majority of the programs I write, and I write them just to try out one idea or another. Occasionally though, I will produce a large and significant program, the best of which I have listed above. I typically have two or three large programs in progress at any given time that I come back to again and again. The main programming language I use for these projects is Blitz 2D/3D, which is a game engine. As such, it is good for graphical stuff, although in terms of programming flexibility, it is definitely lacking compared to Python or Lua.

ProjectEuler is a hobby of mine, and at the moment, I have 112 problems solved, 18 of which are after the first 100. ProjectEuler was largely responsible for the programming knowledge and experience I amassed as the problems require both math and programming to solve. The parameters are too large to brute-force, which is why math comes in, and the parameters are also so large as to make human hand-calculation effectively impossible, thus a computer is required to do the calculations. Through this website, I taught myself a year's worth of college-level programming and more. Python is the main language I use to solve these problems as it is my "number crunching" language, which unfortunately also makes it inferior to languages such as Blitz 2D/3D when it comes to graphical stuff.
